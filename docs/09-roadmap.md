# Roadmap

Roadmapen under beskriver faktisk ønsket retning for repoet. Den erstatter eldre 2.0- og redesign-notater.

## Fase 1: Spillbibliotek og gameplay polish

Mål:

- rydde biblioteket og prioritere de sterkeste spillene
- forbedre spillkort, tempo og lesbarhet
- gjøre anbefalinger og kjernevalg tydeligere

Typiske oppgaver:

- bedre kuratering i `game-library.ts`
- videre opprydding i overlappende spill
- små og målrettede forbedringer i korttekster og deck-struktur
- bedre konsistens mellom bibliotek, temaer og `SPILL_OVERSIKT.md`

## Fase 2: Running rules og bedre gameplay

Mål:

- gjøre running rules mer nyttige i faktisk spill
- forbedre gameplay-rytme og tydelige toppunkter
- videreutvikle impact moments bare der det gir mening

Typiske oppgaver:

- finjustere regelkategorier og varighet
- løfte flere deck som faktisk passer running rules
- forbedre pre-game og overgangene inn i spilling
- eventuelt styrke shared-screen-lesbarhet i gameplay-shell

## Fase 3: SEO og distribusjon

Mål:

- gjøre GameNight lettere å finne
- koble artikler, temaer og spill sterkere sammen
- videreutvikle huber og støtteinnhold uten å miste produktfokus

Typiske oppgaver:

- nye temaer og artikler der det finnes tydelig søkeintensjon
- bedre internlenking mellom spill, temaer og artikler
- vedlikeholde metadata, sitemap og trust-sider
- vurdere hvilke sider som faktisk bør indekseres

## Fase 4: Monetisering

Mål:

- gjøre produktet mer bærekraftig uten å ødelegge opplevelsen

Typiske oppgaver:

- ferdigstille donasjonsflyt mot ekstern tjeneste
- vurdere annonser på passende flater
- sikre compliance, consent og policy-egnethet
- holde gameplay så rent som mulig

## Bevisst utenfor scope

Dette er ikke del av nåværende roadmap med mindre det bestilles eksplisitt:

- multiplayer mellom flere enheter
- kontoer og innlogging
- database og adminsystem
- full redesign av hele produktet
- større omskriving av spillmotoren uten tydelig behov
